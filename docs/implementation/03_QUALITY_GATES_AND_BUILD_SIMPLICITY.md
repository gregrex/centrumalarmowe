# Quality gates and build simplicity

## Cel
Kontynuacja planu wdrożenia z naciskiem na grywalność, jakość grafiki 2D i prostotę produkcji.

## Bramy jakości
### Gate A — playable clarity
- gracz rozumie priorytet w mniej niż 1 sekundę,
- operator może wykonać najważniejszą akcję w mniej niż 2 tapnięcia,
- brak nieczytelnych alertów nakładających się na siebie.

### Gate B — build simplicity
- nowy scenariusz można dodać bez zmiany kodu klienta,
- nowe ikony i portrety można dołączyć przez katalog assetów,
- nowe słowniki przechodzą walidację skryptem.

### Gate C — art quality
- assety przechodzą test miniatury,
- mapa nie gubi czytelności w telefonie,
- każdy stan alarmowy ma własny silny sygnał wizualny.

### Gate D — bot reliability
- bot potrafi obsłużyć rolę przez pełną sesję,
- log bota jest czytelny,
- gracz może przejąć rolę bez rozpadu sesji.
