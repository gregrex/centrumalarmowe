# V22 — Release candidate showcase scope

## Cel
Przygotować pierwszy release candidate jednej misji showcase tak, aby agent mógł:
- domknąć prezentacyjny vertical slice,
- przejść przez bug bash,
- wygenerować release notes,
- przygotować Android APK/AAB mock pipeline,
- spiąć grafikę, sceny, dźwięki i UX pod pokaz produktu.

## Zakres v22
1. Release candidate checklist i gates.
2. Bug bash flow i kategorie defektów.
3. Draft release notes i build naming.
4. Android APK/AAB mock pipeline.
5. Final showcase media pack: screenshots, preview shots, capsule mocks.
6. RC polish dla onboardingu, runtime, raportu i retry/next.
7. Handoff do review build, test deploy i prezentacji.

## Główne bramy jakości
- Jedna misja showcase ma kompletny flow od Home do report room.
- Bug bash ma jasne severity i owner flow.
- Release candidate ma gotowy draft release notes i identyfikator builda.
- Android pipeline ma jasno rozpisane wejścia, artefakty i walidacje.
- Player-facing UX jest czytelny na telefonie pionowym 9:16.
