# Enums, keys and ids

## Reguły nazewnictwa
- małe litery,
- separator `.` dla domeny,
- brak spacji,
- brak polskich znaków w kluczach,
- nazwy użytkowe tylko przez lokalizację.

## Przykłady
- `hud.alert.critical`
- `mission.tutorial.01`
- `zone.center.market`
- `bot.profile.safe_dispatch`
- `weather.heavy_rain`

## Zakazy
- nie używaj surowych stringów w Unity UI,
- nie duplikuj tych samych statusów w kilku plikach,
- nie mieszaj enumów technicznych i contentowych.
