# Final polish — grafika, audio, sceny

## Grafika
- kontrastowe markery stanu,
- grubsze linie tras przy wysokim stresie,
- mniej drobnicy tła w runtime,
- większe strefy dotyku na kartach decyzji.

## Audio
- lżejszy mix tła podczas recovery cards,
- wyraźne potwierdzenie assign/reroute,
- krótkie stingers dla success/partial/fail.

## Sceny
- menu i runtime mają czuć się jak jeden produkt,
- report room ma być nagrodą wizualną, a nie tylko listą wyników.
