# Stany UI i alerty

## Stany globalne
- idle,
- incoming,
- active incident,
- escalation,
- overload,
- system degraded,
- reconnect,
- bot takeover.

## Alerty
- informacyjny,
- ostrzegawczy,
- wysoki,
- krytyczny,
- globalny.

## Stany ekranów
- loading,
- partial sync,
- stale data,
- success,
- warning,
- error,
- blocked action.

## Kluczowe reguły
- alert krytyczny ma zawsze być widoczny ponad lokalnym stanem ekranu,
- bot takeover musi być wyraźny, ale nie zasłaniać rozgrywki,
- reconnect musi dać szybki powrót do sesji.
