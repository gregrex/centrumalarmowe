# Operator i Dispatcher — realny HUD v7

## Cel
Podnieść grywalność przez ograniczenie liczby ruchów potrzebnych do wykonania krytycznych decyzji.

## Operator HUD
### Sekcje
- top alert bar,
- active call stack,
- szybkie tagi klasyfikacji,
- skrót do listy aktywnych incydentów,
- bot takeover badge,
- dolny pasek szybkich akcji.

### Zasady
- maksymalnie 3 główne CTA naraz,
- czerwony alert zawsze ponad wszystkim,
- timer incydentu widoczny bez scrolla,
- teksty krótkie, bez medycznych ścian tekstu.

## Dispatcher HUD
### Sekcje
- active incident board,
- live unit roster,
- map filters,
- route preview card,
- shared action panel,
- timeline feed.

### Zasady
- wybór jednostki i route preview w jednym flow,
- filtry muszą działać jednym tapnięciem,
- lista jednostek pokazuje tylko najważniejsze statusy: free, busy, delayed, unavailable,
- jeśli ETA jest wysokie, gra musi ostrzegać kolorem i ikoną.

## Mobile readability
- touch target minimum 44x44,
- brak małych checkboxów,
- tags i severity jako chipsy,
- ważne liczby z grubszym fontem,
- linie tras bez przesadnych ozdobników.
