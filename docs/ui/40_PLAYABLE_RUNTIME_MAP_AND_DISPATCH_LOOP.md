# Playable Runtime Map And Dispatch Loop

## Ekran główny runtime
- centrum: mapa miasta 2D,
- lewy panel: aktywne incydenty,
- prawy panel: jednostki i cooldowny,
- górny pasek: city pressure, morale, objective state,
- dolny pasek: quick actions dispatchera.

## Core interactions
1. Tap incydent.
2. Podgląd szczegółów i objective impact.
3. Wybór jednostki.
4. Podgląd ETA, ryzyka i zajętości.
5. Potwierdzenie dispatchu.
6. Wynik idzie do event feed i objective state machine.

## Touch UX
- duże cele dotykowe,
- max 2 tapy do dispatchu,
- kolor i ikona muszą komunikować pilność,
- route overlay musi być czytelny na telefonie.
