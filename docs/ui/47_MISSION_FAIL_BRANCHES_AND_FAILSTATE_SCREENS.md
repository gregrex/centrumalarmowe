# Mission Fail Branches and Failstate Screens

## Cele
Porażka ma być czytelna, uczciwa i użyteczna.

## Typy fail branch
- objective critical fail,
- timer fail,
- cascading overload,
- wrong prioritization,
- city pressure collapse,
- team readiness breakdown.

## Ekran failstate
- główny powód porażki,
- timeline ostatnich zdarzeń,
- 3 najważniejsze błędy,
- rekomendacja poprawy,
- CTA Retry,
- CTA Return to Briefing,
- CTA Home.

## Wymagania UX
- zero chaosu,
- mocny kontrast,
- oszczędny tekst,
- szybka nawigacja do retry.
