# Test build session stability and fallbacks

## Zakres
- offline showcase,
- online review room,
- bot fill fallback,
- rejoin do runtime,
- retry po utracie połączenia.

## Wymagania
- brak crasha przy utracie jednego gracza,
- bot fill może przejąć rolę w review build,
- retry flow nie gubi stanu misji demo,
- capture mode działa niezależnie od trybu sesji.
