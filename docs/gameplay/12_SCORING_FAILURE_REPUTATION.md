
# Scoring, failure, reputation

## KPI rdzeniowe
- response_time_ms,
- dispatch_time_ms,
- incidents_resolved,
- incidents_failed,
- incidents_escalated,
- unnecessary_dispatch_count,
- player_disconnect_takeovers,
- average_pressure.

## Ocena końcowa
- S: kontrola sytuacji, mało eskalacji, szybka reakcja,
- A: bardzo dobrze, drobne błędy,
- B: stabilnie, ale z opóźnieniami,
- C: słaba kontrola, dużo eskalacji,
- D: system przeciążony,
- F: utrata kontroli nad rundą.

## Porażka
Runda nie musi kończyć się natychmiast game over. Najpierw:
- fail local incident,
- fail public order,
- fail overload threshold,
- fail full-session.

## Reputacja
Reputacja działa między rundami i wpływa na:
- dostęp do lepszych scenariuszy,
- odblokowanie lepszych narzędzi,
- trudność eventów medialnych,
- intensywność presji społecznej.
