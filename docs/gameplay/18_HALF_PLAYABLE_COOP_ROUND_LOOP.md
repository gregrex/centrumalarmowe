# Półgrywalny loop rundy coop

## Cel
Dostarczyć pierwszy odczuwalny loop grywalności bez pełnej produkcji.

## Przebieg
1. Start rundy.
2. Pojawia się 3–5 incydentów.
3. Operator odbiera alarmy i oznacza priorytet.
4. Dispatcher wybiera jednostkę.
5. Overlay pokazuje trasę.
6. Coordinator potwierdza shared action dla krytycznego incydentu.
7. Crisis officer może przesunąć priorytet miasta lub poprosić o wsparcie.
8. System generuje delty.
9. Boty przejmują brakujące role.
10. Runda kończy się częściowym raportem.

## Success
- właściwa obsługa większości incydentów,
- mało timeoutów,
- mało błędnych dispatchy.

## Failure
- za dużo timeoutów,
- incydent krytyczny bez shared action,
- przeciążenie wszystkich jednostek.
