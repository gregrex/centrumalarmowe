# Runtime HUD, Recovery and Retry Audio

## Warstwy
1. Runtime base loop.
2. Alert stack.
3. Recovery stingers.
4. Objective transition cues.
5. Fail / retry / next stingers.

## Zasady
- recovery cue musi być wyraźny, ale krótki,
- retry cue ma dawać energię, nie karę,
- fail cue ma być poważny, ale nie przytłaczający,
- success cue ma być krótki i profesjonalny.

## Priorytety
critical alarm > mission voice bark > recovery cue > UI click > ambience
